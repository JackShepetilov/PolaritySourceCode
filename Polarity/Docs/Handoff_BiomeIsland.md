# Handoff: биом «остров-цитадель» (брифинг для нового чата)

> Самодостаточный бриф по итогам дизайн-сессии 2026-06-10 (все решения — автора). Схема: `Docs/BiomeStructure_v4.svg`. Правила арен/метрики: `Docs/LevelDesign.md`. Биом расставляет автор; арены = готовые сублевелы (см. `Handoff_ArenaSublevels.md`).

## Структура биома (финал v4)

- **Остров-цитадель**: профиль треугольный/трапециевидный. От «плеча» высадки (40–50 м над водой) — длинный подъём к пику на стороне старта; на пике, на краю **обрыва**, — цитадель с ключевой hand-made ареной (**150–200 м**). Ландмарк на ближнем к старту углу → **прямая видимость со старта рана и со всей спирали**.
- **Вход в ран — обязательная череда «мальдив» по спирали ПРОТИВ ЧАСОВОЙ** вокруг острова: микро-острова с S-аренами (тир 1). **Число мальдив произвольное — решает автор** (не фиксировано на 3–4). **Высоты низкие: от ~5 м у старта до ≤30 м к высадке** (интерполяция по цепочке) — вертикальная драма набирается подъёмом по острову, не по мальдивам (выбран «вариант А»: лёгкий набор высоты по спирали, высадка в нижнюю зону острова).
- **На острове 3 generic-арены-стражи** (тир 2–3) на склоне, **на немного разной высоте, свободный порядок** (не плато, не цепочка).
- **Гейт цитадели**: диегетический счётчик антенн на воротах (3/3 со стражей-арен; вклад мальдив — TBD). Антенна любой арены = level-up (готовая механика).
- Босс — за цитаделью (hand-made всегда) → следующий биом.
- Смысл сборки: спиральный линейный вход = контроль пейсинга начала рана (Void Breaker/Roboquest) + showcase ландмарка со всех ракурсов; открытая середина = свобода/реиграбельность; ландмарк = ориентация без UI и «постер» биома.

## Решения по системам (приняты автором)

- **Границы боя — «стакан»**: невидимых стен нет; на аренах — полупрозрачные glass-поля по периметру до ~25–30 м, включаются ТОЛЬКО на бой через `ExitBlockers` ArenaManager (уже реализовано во всех 5 аренах; запирают и дронов — была проблема вылетов). У цитадели — постоянное поле до 3/3 антенн; альтернатива жёсткой стене — отталкивающий импульс (LaunchCharacter от нормали) — A/B-тест. Честность: граница по читаемой геометрии, вспышка при касании, счётчик на воротах.
- **Сферические блокеры мёртвых** (мешали, выкинуты автором ранее) — не возвращать.
- **Переходы-вэлвы** между мальдивами/аренами: 15–30 с движения (~5 000–15 000 uu при 1000–1500 cm/s), движенческие мини-задачи (wallrun-цепочки, прыжковые дорожки, тросы), карманы риск/награда. Падение в воду = потеря времени, не прогресса (выплыл к лесенке).
- **Вода не смертельна** (плавание появится позже), kill-объёмов нет. Вода = граница мальдив. Свободный рим по острову разрешён: дорожки быстрые (вложения сюда), целина медленная, 2–3 секрета на биом; навмеш/AI только в аренах.
- **Гейтинг для новичка vs ветерана**: до первой победы над боссом биома гейт цитадели жёсткий (счётчик превращает запрет в цель); после победы — снимается навсегда (мета-флаг; лор: «друг взломал ворота»). Движенческие шорткаты маршрута ≠ доступ в цитадель.
- **Без гуманоидов в биоме 1** — только роботы.

## Процедурность/рандомность (решение: вариант «один каркас + слои рандома»)

Остров-каркас один (ландшафт, спираль, дорожки, цитадель — фиксированы). Рандомизируется то, что игрок чувствует:
1. Раскладка пула арен по слотам (слот = transform + тир; арены — самодостаточные сублевелы в локальных координатах, главный вход с −Y, грузятся в рантайме `LevelStreamingDynamic::LoadLevelInstance` с трансформом).
2. Зеркалирование арен (флаг `mirror` в билдере: x→−x) — удваивает пул бесплатно.
3. 2–3 пресета волн на арену (`wave_sets` в JSON — добавить в билдер).
4. Тасование переходов-вэлвов между слотами (вэлвы — тоже маленькие сублевелы, 6–8 шт).
5. Рандом точки старта спирали/направления.
Нужен `BiomeLayoutSubsystem` (~150 строк: сид рана → раскладка) — НЕ написан; C++ или BP — спросить автора. Полностью процедурный ландшафт отвергнут (валидация правил дороже эффекта). Второй лейаут острова («биом 1б») — когда пул арен дорастёт до 8–10.

## Чем заканчивается обсуждение / следующие шаги

1. Пейпер-макет биома в масштабе (слоты 3 арен на склоне, N мальдив спирали — число согласовать с автором — с бюджетами переходов, позиция цитадели/обрыва) — фаза 0, SVG.
2. Тестовый сублевел одного спирального перехода (вэлв) + «стакан» цитадели с прототипом отталкивающего поля.
3. `BiomeLayoutSubsystem` (слоты/пулы/сид).
4. Открытые вопросы автору: вклад мальдив в гейт (3/3 только стражи?); пути отладочных сублевелов для `extra_sublevels`; C++ или BP для layout-системы.

## Рельеф v8 — модель «дороги + точные пады» (2026-06-12, смузинг-сессия)

Полная переработка геймплейных штампов в `make_biome1_heightmap.py` (слоты НЕ двигались; жалобы автора: угловатость, плиссе у G2, ландшафт наслоился на амфитеатр). Что теперь является контрактом рельефа:

1. **Дороги вместо конусных врезок.** Серпантин = градуированная лента (ядро 380 uu, профиль ≤26.6° по построению, врезка И насыпь): огибающая по сглаженному рельефу + жёсткие якоря высот, между якорями провалы перекрываются хордой (дамба). Стенки врезок: C2-фейд на мелких перепадах, чистая терраса ≤32° на глубоких (переключение по |Δh|, порог 700–1400). Ноги выводятся из слотов автоматически: legA = shoulder → G1 → G2 → вход A5 (вершины route_corridor = жест автора), legB = полка за bort_s → посадка лестницы цитадели. Внутри апронов профиль прибит якорями на кромках (apron_edge_point).
2. **Пады G1/G2 по футпринту арены** (повёрнутый прямоугольник из спеки + 500 uu, smootherstep 3400), высота = уровень ОПОРНОЙ ПЛИТЫ арены: ground = min(floor top ≤ 100) из спеки. A7_Gallery несёт outer_ground на −500 → апрон G2 = slot_z − 520, подиум возвышается как задуман. Воротники падов — та же конструкция fade/terrace.
3. **Чаша G3/A5 — точная, в локальном фрейме арены**: капы под террасами (0/−350/−700 минус клиренс 150; ступень капа на 150 uu ВНУТРЬ верхней террасы — иначе билинейные щели у кромок), контактные полосы входа/боковой двери/выходной полки (плита − 30), воронка ≤32° снаружи. Выход игрока к цитадели: антенна → перемахнуть bort_s (250) → полка → дорога 1.8k к посадке лестницы. **Капы чаши переутверждаются ПОСЛЕ блока лестницы** — её разглаживание посадки (r 1700–3400) дотягивается до юго-восточного угла чаши и иначе хоронит T2 (+390; это и была «наслойка»).
4. **Берег**: C2-ремап градиента (smootherstep-вес, полоса |H−60|<420) вместо жёсткой полки — угловой уступ на h≈110 ушёл. **Глобально**: curvature_relax (блюр, взвешенный |лапласианом|) по проходимой земле и финальный полиш швов по полосам дорог/воротников/кольца чаши (ядра дорог и контакты исключены, капы переутверждаются).
5. **Лестница цитадели — без изменений** (тот же блок, применяется последним, верификация полётов/посадки в каждом прогоне).

**Вариации (roguelike)**: `python make_biome1_heightmap.py --seed N` — сид меняет ТОЛЬКО природные слои (хребет, варп, фазы побережья/мальдив); пиллары (мыс-цитадель, ось спавна, цепь мальдив, дуга стражей, маршрут, штампы, проверки) прибиты к layout JSON и не зависят от сида. Выход с не-дефолтным сидом: `biome1_heightmap_2017_seed<N>.png`. Перед использованием варианта — прогнать `analyze_island_terrain.py <png>`: он проверяет рельеф против РЕАЛЬНОЙ геометрии всех 8 арен (повёрнутые футпринты, «BURIED»-репорт), углы маршрута игрока и складки. Слоты двигать как раньше (sync_island_slots.py): ноги/контакты пересчитаются сами; при невыполнимом подъёме (>0.50 между якорями) генератор громко ругается «INFEASIBLE».

**Гейты в генераторе** (каждый прогон): профили ног ≤26.6°, маршрут по финальной поверхности 0 сэмплов >30°, A5 T0/T1/T2 чисто, пады/контакты точно, лестница OK. In-engine после сборки: пробы ensure_landscape (G1 апрон = slot−20, G2 = slot−520, чаша G3 = slot−490).

**Мальдивы = «песчаные линзы» (2026-06-12, после двух итераций с автором).** Плоским остаётся только футпринт арены (плиты висят в 10 uu над ним); от его кромки остров падает выпуклым C2-плечом сразу под воду (`run = z·2.8 + 2000`, виден как `MALDIVE_RUN_K/BASE` в генераторе) — ни плоского кольца, ни вала по кромке (оба варианта автор отверг: «фишка казино», «пепельница»). Дюнный шум растёт с падением (база 20 сразу за футпринтом, до ±140 на склоне), 2θ-гармоника вытягивает островки в овалы. M1 сливается со стартовым рифом песчаной косой — осознанно (тосс летит поверх, вода обязательна только МЕЖДУ аренами). ⚠️ Для арт-пасса: `make_biome1_weightmaps.py` ещё считает зоны/пальмы по СТАРОЙ радиальной модели (`MALDIVE_SHORE_RUN`, депрекейтнута) — кольца пальм и veg-исключения надо пересинхронизировать на линзу (пальмы по старым радиусам местами окажутся под водой/на склоне).

## Layout Editor — авторская тулза острова (2026-06-12)

`python Source/Tools/ArenaBlockout/layout_editor_server.py` → браузер `http://127.0.0.1:8765`. Автор сам двигает арены (драг, yaw-стрелка = вход), большой остров (центр/радиус/пик/купол/сектор утёса/вырезы берега), точки серпантина (dblclick — добавить, Del — удалить; точки у арен анкерятся к падам), пер-островные линзы мальдив (ромб = направление хвоста; длина/изрезанность/размер), сид и лимит уклона дорог. Кнопки: **Fast preview** (~5–10 с, 505², гейты приблизительные — квантование даёт ложные +50..80 на A5/A7), **Full** (2017², точные гейты), **Apply → UE** (активирует лейаут, полный прогон, сборка в открытом редакторе через REST — статус-точка справа), **Sync из UE** (подтянуть позиции арен из редактора). Именованные лейауты в `Island/layouts/*.json` (New/Del); Apply копирует выбранный в ACTIVE (`biome1_island_layout.json`) с бэкапом в `Island/editor_work/backups/`. Артефакты тулзы (`editor_work/`) в git не ходят.

Технически: вся «умность» осталась в `make_biome1_heightmap.py` (новые параметры `--layout/--res/--out/--preview`, сид персистится в поле `seed` лейаута; маршрут обобщён — `derive_route()` строит ноги из произвольных вершин коридора, и его же импортирует анализатор, так что проверки не могут разъехаться с билдером). Полный прогон без изменений лейаута даёт байт-в-байт прежний heightmap (регрессия проверена).

**Железное правило генератора (поймано в движке 2026-06-12): НИКАКИХ бинарных масок на операциях с высотами.** Потексельное «внутри/снаружи» между поверхностями, отличающимися на 100+ uu, рендерится рядом пирамид вдоль повёрнутой границы (полосы контакта × `dist>0`), а бинарное «не трогать ядро дороги» в штампе воротника — пилой на стенках выемки. Все веса — гладкие (`smoothstep` по расстоянию); капы согласуются с контактами лифтом (`max`), а не порядком применения; ступени капов прячутся в телах плит/стен гладкими рампами. Финальная страховка — despike-проходы (одиночные тексели против медианы 8 соседей) + гейт `spikes: 0` в `analyze_island_terrain.py`. Бинарный термин допустим только там, где он гейтит математически совпадающие на границе значения (внутренность апрона при весе штампа, уже равном 1).
