# POLARITY — Правила левелдизайна арен (v0.1, draft)

> Рабочий документ для блокаутов generic-арен. Основан на: `GameDesign.md`, `CombatMechanics.md`, фактах из кода (Source, июнь 2026), ресёрче Titanfall 2 (action blocks, wallrun-геометрия, разборы Effect and Cause) и VOID/BREAKER (экономика деструкции, читаемость, ритм ран-нод).
>

## 0. Структура биома (актуальная)

Биом = остров. На острове:
- **5 generic-арен** (блокауты по этому документу) — каждая с антенной: зачистил арену → активировал антенну → level-up.
- **1 hand-made арена** (автор) — на центральном возвышении острова, максимум лора о злодее.
- **Босс-арена** (hand-made) — открывается после зачистки нескольких generic + центральной арены, ведёт в следующий биом.

Биом 1 — остров по мотивам Little Saint James.

> Прим.: §0 в `CombatMechanics.md` («4 тира × 3 арены») описывает старую структуру — при случае обновить.

**Разделение зон ответственности (решено 2026-06-10):** Claude делает арены как **отдельные сублевелы в локальных координатах** (центр чаши ≈ (0,0,0)); расстановку сублевелов на острове и сам биом делает автор. Правила §1 — рекомендации автору для расстановки, всё остальное — рабочие правила арен.

## 1. Макро-правила острова (зона автора)

**M1. Иерархия ориентиров.** Три уровня: центральная гора с hand-made ареной (видна отовсюду) → антенны арен (вертикальные доминанты с beacon-VFX, видны с маршрутов) → локальные ориентиры внутри арен. Игрок в любой точке острова должен без карты понимать, где центр и где ближайшая незачищенная арена. `AArenaAntenna` уже имеет Niagara-beacon, который разгорается после зачистки — до зачистки антенна должна читаться силуэтом.

**M2. Ритм «бой → выдох → выбор» (VOID/BREAKER).** Переходы между аренами — передышка: без врагов (или с одиночными), с лором, секретами, видами на следующие арены. Длина перехода 15–30 с движения (~5 000–15 000 uu при характерной скорости 1 000–1 500 cm/s). Короче — арены сливаются, длиннее — провисает темп.

**M3. Подход к арене = превью.** С тропы подхода арена читается целиком: где вход, где антенна, какая вертикаль. Правило TF2: игрок планирует маршрут в первую секунду. Вход — возвышенный или открытый «порог» с обзором чаши.

**M4. Контур арены под ExitBlockers.** На время боя `AArenaManager` включает блокеры выходов. Геометрия каждой арены обязана иметь явные проёмы-входы (2–3), куда встают блокеры (ворота/силовое поле), и непроходимый периметр везде, кроме них (скалы, стены, перепад высот). Никаких «дырявых» периметров — иначе блокировка выглядит произвольной.

**M5. Край острова.** Вода НЕ смертельна (механика плавания/выхода на берег появится позже), kill-объёмы НЕ ставим (решено 2026-06-10). Но борта остаются геймплейным правилом: боевые зоны не граничат с открытой водой; между боем и водой — борт ≥ 150 uu (парапет, скалы, контейнеры). Причина: дропы оружия и пикапы (магнит 500 uu, lifetime 15 s) тонут безвозвратно, а вылет игрока в воду на скорости 1 500+ cm/s ломает темп боя.

**M6. Распределение арен.** Generic-арены — по периметру/секторам острова, каждая в своём «биом-фрагменте» (пляж, причал, постройки, склон...), чтобы силуэты различались. Сложность растёт к центру/боссу: ближние к спавну арены — S-размер и простой состав, дальние — L и элитный состав.

## 3. Метрики врагов, влияющие на геометрию

| Факт | Следствие для ЛД |
|---|---|
| Боевой круг: Inner 400–600, Middle 600–1200, Outer 1200–2000 [код/док] | Ядро арены должно вмещать окружность Ø ~4000–5000 uu вокруг подвижного игрока — иначе слоты колец вырождаются |
| Стрелок: отступает при <250 в течение 1.5 s, держит 600–1200 | Нужно пространство отступления за спинами стрелков; не зажимать их в ниши |
| Самурай: наземный, дэш, магнетизм 600 | **Обязана существовать наземная связность**: на любую позицию игрока (кроме скилл-перчей) есть путь по навмешу с рампами ≤ walkable slope; иначе кемп |
| Дрон: ховер 200–450 над землёй, clearance от потолка 100, EngageRange 2000 | Потолки в зонах дронов ≥ 600 uu; дроны проектируют цели на навмеш → навмеш должен покрывать и «дроновые» зоны |
| Камикадзе-дрон: орбита+пике, рой | Открытые объёмы для дуг пике; не запускать в низких интерьерах |
| Турель: наводка 2.0 s [код] / 3.0 s [док], поворот 90°/s, pitch +45/−30 [код] | Турель держит **одну длинную линию**; мёртвая зона у основания: при высоте H не видит ближе ~1.7×H [расчёт]. Подход к турели = фланг или нижняя слепая зона — закладывать оба |
| Гуманоид: yank-цель, после разоружения — melee-раш | Как самурай: наземная связность; плюс зона «дуэли» средней дистанции для фазы щит/оружие |
| Wallslam: порог 800 cm/s [код] | Стены = оружие. Хотспоты боя в ≤ 1000–1500 uu от slam-поверхности |
| Взрыв пропа: r 300, стан 2 s; дроп HP | Пропы кластерами 3–5 у хотспотов, не в центре арены |
| Пикапы: магнит 500, живут 15 s | Полы хотспотов — «ловящие»: без щелей, дыр, открытых краёв |

**Экономика заряда:** заряд игрока копится в ближнем бою и тратится на канализацию/способности. Правило состава: первая волна каждой арены содержит melee-доступные цели (самураи/пехота близко к входу), иначе игрок входит «пустой» и петли EMF голодают.

**Состав биома 1: БЕЗ гуманоидов** (указание автора 2026-06-10) — только ИИ-роботы (стрелки, самураи, дроны, камикадзе, турели). Органика появится в поздних биомах.

**Клиренс спавн-поинтов: ≥ 250 uu до любой коллизии** (включая bounds повёрнутых объектов!) — иначе NPC спавнятся упёршись в меш и ломается агро/навигация (урок A1, 2026-06-10).

## 4. Двенадцать правил арены

**R1. Чаша с петлями.** База арены — «боевая чаша» с ≥ 2 пересекающимися кольцевыми маршрутами (минимум — восьмёрка). Каждый маршрут проходим на скорости без остановок. Тупики глубже 600 uu запрещены (исключение — ниша с наградой, простреливаемая насквозь).

**R2. Три слоя.** L0 — пол (мили, пропы, основной бой); L1 — +300–450 (балконы/контейнеры/террасы — доступны прыжок+мантл и wallrun); L2 — +700–1000 (крыши/перчи — wallrun-цепочки, пластины, rocket boost). Урок TF2: плоские арены в 1–2 слоя убивают движение. Слои связаны минимум в 3 точках каждый. **Связность для NPC = NavLinkProxy-прыжки (обязательны на каждом перепаде слоёв) + рампы как опция.** NPC прыгают между ярусами по линкам, а не бегут к лестнице — поэтому допустимы арены вовсе без лестниц, с разрывными поверхностями между этажами (решение автора 2026-06-10).

**R2a. Nav links обязательны (решено 2026-06-10).** На каждом перепаде слоёв — `NavLinkProxy`: NPC **прыгают** между ярусами, а не бегут к лестнице. Рампы — для читаемости маршрута игроком и как запасной путь, но связность AI держится на линках. Следствие: допустимы арены вообще без лестниц, с разрывными поверхностями между этажами — если линки покрывают все переходы. Линки ставить в местах без парапетов, по 2+ на каждую грань яруса.

**R3. Никаких неуязвимых насестов.** Любая позиция, куда наземный NPC не дойдёт, должна простреливаться турелью ИЛИ быть в зоне дронов/камикадзе. Высота = позиционирование, не безопасность (VOID/BREAKER: враги лезут за игроком).

**R4. Wallrun-стены — святое.** Гладкие, монолитные, без выступающих коллизий (трим/трубы/карнизы — только no-collision декор поверх плоского блокинг-объёма). Длина 800–1600, высота ≥ 250 (чтобы читалась), помечены единым материалом/цветом блокаута. Стоят **вдоль** маршрутов-хайвеев и поверх опасных зон (над пропастью причала, над «озером» турельного сектора), а не перпендикулярно потоку. Главная жалоба на TF2-карты — стены с буграми и невидимые маршруты: у нас один визуальный код и ноль исключений.

**R5. Хайвеи и перекрёстки.** 2–3 скоростных маршрута (земля/L1/wallrun-лента) пересекаются в 1–2 контестед-точках с центральным объектом (мост, колонна, ангар). Бой стягивается к перекрёсткам, движение — по хайвеям. Лейны — широкие «дороги» (ширина 600–1000 uu, потолок ≥ 600), а не снайперские коридоры: ритм укрытий каждые 600–900 uu вдоль любой линии длиннее 3000.

**R6. Сайтлайны под контролем.** Максимальная диагональ простреливания ≤ 6000–8000 uu и таких линий не больше 1–2 — и именно на них сажаем турель (она и есть «владелец» длинной линии). Остальные углы обзора рубятся на 2000–4000. Ни одна точка входа врага не видна со всех точек арены — и наоборот, игрок со входа видит **всю композицию** (обзор — да, прострел всего — нет).

**R7. Укрытия двух высот, обе проходимые.** Низкое 100 (перепрыгивается, слайд под огнём), полное 200 (мантлится). Укрытие никогда не прерывает поток: любое укрытие можно взять с разбега (мантл/прыжок), а не оббегать. Расстановка — ритмом вдоль маршрутов, не «островками посреди поля».

**R8. Стены-оружие у каждого хотспота.** В радиусе 1000–1500 uu от любого места, где затевается бой, есть slam-поверхность (стена/колонна/контейнер) и кластер из 3–5 EMF-пропов. Пропы — у краёв хотспота (запуск «сквозь» группу врагов), не в геометрическом центре арены (анти-clump урок VOID/BREAKER).

**R9. Спавны по периметру и под-пространствам.** Точки `AArenaSpawnPoint` — за пределами текущего поля зрения с основных маршрутов, ≥ 1500 uu от вероятной позиции игрока, разбиты на 2–3 группы по под-пространствам (TF2: 11–15 врагов сплитом, не блобом). Air-spawn точки дронов — на 300 uu, над L1/L2, не над головой игрока. Спавн в чистое поле запрещён — каждой группе своя «кулиса» (за зданием, из-за контейнеров, из проёма).

**R10. Антенна — финал, не центр.** Антенна стоит на возвышении у края чаши (L1/L2), видна из любой точки боя и с подхода, но кнопка активации — короткий «выдох»-маршрут после зачистки (10–15 с подъёма), с видом на остров/следующую арену. Не прятать; не ставить в гущу боя.

**R11. Читаемость прежде всего.** Блокаут-палитра фиксирована: пол — серый, стены — тёмно-серый, wallrun-поверхности — один акцентный цвет, EMF-элементы (пластины/заряды) — второй акцент, опасное (DamageVolume, края) — третий. Никакого визуального шума в воздушных объёмах боя: главный провал жанра — «не видно врагов за эффектами» (главная претензия к VOID/BREAKER). Силуэт каждой арены различим с воды и с центра острова.

**R12. Один экран — один план.** Со входа игрок за секунду отвечает на три вопроса: куда бежать (хайвей), где опасно (турель/группа), где профит (пропы, антенна). Если для ответа надо остановиться — композиция провалена. Проверка: скриншот со входа без UI понятен постороннему.

## 5. Пять архетипов generic-арен (биом 1)

Ярлыки рабочие (не названия!). Размеры — боевое ядро без периметра. Все арены: 2–3 входа под ExitBlockers, антенна по R10, спавны по R9.

**A1 — «Причал» (S, интро).** 4000×6000, доминанта — линейный пирс с контейнерами. Два слоя: настил + крыши контейнеров (+300). Контейнерные стены = wallrun-лента вдоль всей длины + slam-поверхности везде. Состав: пехота+самураи, 1 волна дронов. Учит: wallrun-хайвей, слам в контейнер, борта у воды (M5). Простейшая восьмёрка вокруг двух контейнерных блоков.

**A2 — «Двор виллы» (M, эталон чаши).** ✅ Одобрена автором (2026-06-10). Центральный объект чаши — хороший приём (LOS-блокер + точка для арта: памятник/фонтан), но НЕ обязательное правило — применять по месту. 5000×5000 двор + балконы по периметру (+350, рампы с двух сторон), центральный объект (фонтан/беседка) как перекрёсток и LOS-блокер. Полный боевой круг работает идеально (Ø чаши ~5000). Состав: смешанный + гуманоид-«капитан» на балконе. Петли: кольцо двора × кольцо балконов, связки — мантл-парапеты 200.

**A3 — «Купол» (M-L, вертикаль).** Терраса с храмом-куполом (отсылка к полосатому куполу LSJ — лорная точка биома). Спираль из трёх террас (+0/+400/+800) вокруг купола, wallrun-сегменты по барабану купола. Турель на верхней террасе держит единственную длинную линию (вход→купол); её слепая зона снизу — маршрут штурма. Состав: стрелки+дроны, камикадзе во 2-й волне.

**A4 — «Ангар/серверная» (M, интерьер).** 4000×7000, потолок 800–1000. Ряды серверных стоек = параллельные wallrun-стены (длина ~1200) + slam-поверхности + LOS-ритм. Фланговые техкоридоры (ширина 400). Хайвей по крышам стоек (+250, мантл). Состав: гуманоиды+пехота, без камикадзе (низкие объёмы). Урок TF2-интерьеров: потолок не ниже апекса двойного прыжка ~600.

**A5 — «Амфитеатр на склоне» (L, финальная generic).** 6000×8000, три террасы вниз к воде (−0/−350/−700), длинная диагональ под турелью сверху, EMF-пластины как «лифты» между террасами (вертикальный пинбол), плотные кластеры пропов. Состав: всё сразу, элитные множители. Самая высокая цена ошибки: борт у воды строго по M5.

Прогрессия размещения на острове: A1 у точки старта → A2/A4 в среднем поясе → A3 у центра (лор) → A5 перед боссом.

## 6. Сборка арены (обязательный набор акторов)

⚠️ **Все геймплейные классы переопределены в BP — спавнить/ставить BP-сабклассы, НЕ C++ классы.** Единственное исключение: `AArenaSpawnPoint` используется напрямую (C++). Пути BP:

| Логика | Ассет |
|---|---|
| Менеджер арены | `/Game/Variant_Shooter/Arenas/BPs/BP_ArenaManager` |
| Антенна | `/Game/Variant_Shooter/Arenas/BPs/BP_Antenna` (кнопка активации встроена компонентом; есть и `BP_AntennaToTransition`) |
| Координатор | `/Game/Variant_Shooter/Blueprints/AI/BP_AICombatCoordinator` |
| EMF-проп | `/Game/Variant_Shooter/Blueprints/Objects/BP_EMFProp_1` |
| EMF-пластина | `/Game/Variant_Shooter/Blueprints/Objects/BP_EMF_AcceleratorPlate` |
| NPC (волны) | `/Game/Variant_Shooter/Blueprints/AI/BPs/BP_ShooterNPC`, `BP_MeleeNPC`, `BP_FlyingDrone`, `BP_HumanoidNPC`, `BP_KamikazeDrone`, `BP_TurretNPC` |
| Спавн-поинт | C++ `/Script/Polarity.ArenaSpawnPoint` (телеграф: `BP_TelegraphedArenaSpawnPoints`) |

1. `BP_ArenaManager` — 1 на арену (сублевел): режим Waves (generic: 2–3 волны, 8–15 врагов суммарно) или Sustain; `EntryTriggers[]`, `ExitBlockers[]`; спавн-поинты и антенны автособираются со своего сублевела (`bAutoCollect*FromOwnLevel=true`).
2. `AArenaSpawnPoint` × 8–15 (с запасом; группами по под-пространствам; `bAirSpawn` для дронов; `ExcludedNPCClasses` где надо).
3. `BP_AICombatCoordinator` — 1 на арену.
4. `BP_Antenna` — по R10 (кнопка активации уже встроена в BP, отдельный актор и спаривание не нужны).
5. `BP_EMFProp_1` кластеры (3–5 у хотспотов), `BP_EMF_AcceleratorPlate` (вертикальные связки).
6. **NavLinkProxy на каждом перепаде слоёв** (балкон↔пол, крыша↔балкон): NPC прыгают по линкам вместо обхода через рампы. Линки ставить в местах без парапетов, направление Both Ways. **Обязательно настраивать SMART link** (Smart Link Is Relevant + те же точки в NavLinkCustomComponent): `PolarityPathFollowingComponent` детектит прыжок через `CustomNavLinkId` — голые PointLinks не прыгаются. Отладка: фильтр `[NAV_DEBUG]` в Output Log.
7. PlayerStart (вход), `PlayerRespawnPoint` в ArenaManager (TargetPoint), NavMeshBoundsVolume на всю арену **включая L1/L2 и дроновые объёмы** (дроны проецируют цели на навмеш). Kill-объёмы НЕ нужны (M5).
8. RewardDoor/контейнер — если арена их использует (для антенна-флоу не обязательны).
9. **Освещение НЕ ставить в persistent-уровень арены** — автор добавляет свет отдельным сублевелом. Верификация блокаута — unlit-скриншоты.

## 8. Воркфлоу блокаутов (Claude как левелдизайнер)

**Фаза 0 — спека на бумаге (минуты, без UE).** На каждую арену: top-down схема в масштабе (слои цветом), разрез по главной оси, маршрутные петли, позиции спавнов/пропов/антенны/турели, состав волн. Согласование с автором ДО геометрии — это «action block» из TF2-процесса: дёшево выбросить, дёшево переделать.

**Фаза 1 — данные.** Арена описывается JSON-спекой (`Source/Tools/ArenaBlockout/Arenas/A2_Courtyard.json`): список примитивов (box/ramp с pos/size/rot/тегом-цветом) + маркеры (SpawnPoint/Antenna/Prop/Plate/PlayerStart...). JSON — источник истины, в git.

**Фаза 2 — генерация в UE (решено: Python, плагин включает автор).** Скрипт `Source/Tools/ArenaBlockout/build_arena.py` читает JSON, чистит акторы с тегом `BLOCKOUT_<имя>`, спавнит геометрию (Engine BasicShapes + блокаут-материалы в `/Game/ArenaBlockout/Materials`) и геймплейные BP-акторы (§6), прокидывает связи (волны, триггеры, блокеры, антенна↔кнопка), сохраняет `.umap`-сублевел. Итерация = правка JSON + перезапуск. Два способа запуска:
- **Headless (Claude, автономно):** `UnrealEditor-Cmd.exe Polarity.uproject -run=pythonscript -script="...\build_arena.py A2_Courtyard" -stdout -unattended` (редактор у автора лучше закрыть, чтобы не словить конфликт сохранения).
- **В редакторе (автор, мгновенно):** консоль `py "...\build_arena.py" A2_Courtyard` — собирает прямо в открытой сессии.

**Фаза 3 — верификация.** Скрипт после сборки: (а) дампит фактические акторы+бounds в `Build/<арена>_dump.json`; (б) автоматически снимает **unlit-скриншоты** (камеры из секции `screenshots` спеки: топ-даун, вид со входа, 2 изо) в `Build/Screenshots/` — света в уровне нет, плоские цвета блокаута читаются и без него, Claude сам читает PNG и сверяет с правилами §4. Затем автор делает флайтру/прогон чеклиста §7. Итерация: фидбек → JSON → пересборка одной командой.

**Конвенции.** Карты: `Content/Variant_Shooter/Arenas/Biome1/<ArenaName>/Lvl_<ArenaName>.umap`, каждая арена — отдельный сублевел в локальных координатах (центр чаши ~ (0,0,0)), на остров ставится как streaming level/Level Instance с трансформом. Все сгенерированные акторы тегированы `BLOCKOUT_<ArenaName>` — пересборка идемпотентна. Грид: 50 uu (примитивы), 10 uu (тонкая подгонка).
